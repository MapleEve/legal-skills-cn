# /product-legal:cold-start-interview

## 操作流程

1. 检查实践配置状态。
2. 运行以下冷启动访谈。
3. 种子文件：10份过往产品上线审查文件（从跟踪器或云盘中读取）。全部读取。
4. 从实际被阻止的案例与已发版的案例中构建风险校准表。
5. 迁移：如缓存的配置文件存在则复制到配置路径。
6. 写入实践配置。展示校准表确认。

# 冷启动访谈：产品法务

## 功能目的

产品法务高度依赖公司具体情况。在金融科技公司构成发版阻断项的问题，在广告科技公司只是一般提示。同一功能对被网信办/市场监管总局调查过的公司和从未被监管机构关注的公司，风险完全不同。

本访谈通过阅读您的实际产品上线审查文件来学习您公司的风险校准——您在哪里阻止、在哪里放行、您把时间花在哪里。

## 冷启动检查

读取配置文件，按存在/占位符/暂停/已填充四种状态分派。

## 检查共享公司配置

查找 `~/.claude/plugins/config/claude-for-legal-cn/company-profile.md`：如存在则读取并确认，跳过公司问题。如不存在则第一个设置，询问公司问题并写入共享配置。

## 安装范围检查

如工作目录在项目内而非用户主目录，标注一次。

## 访谈开始前

显示分叉优先的序言（3-4短行）：

> **`product-legal` 适用于审查产品上线、营销文案和功能风险的人员——即发版的法律层面。** 不是您的领域？ `/legal-builder-hub:related-skills-surfacer`。
>
> **2分钟**即可获得您的角色、审查框架级别（正式门禁 vs 咨询）和产品/实践背景（消费者、企业或两者），其余使用合理默认值。**15分钟**增加您的风险校准表（什么被阻止 vs 什么可以发版）、升级矩阵、审查框架类别、备忘录格式和上线跟踪器集成。
>
> 快速还是完整？（随时可用 `/product-legal:cold-start-interview --full` 升级。）

**快速启动路径：** 仅询问第0部分和产品领域。其余使用 `[DEFAULT]` 标注。

## 访谈节奏

- 假设答案存在于某处——提示粘贴链接或文件
- 每条消息2-3个可回答的提示
- 为需要打字的真实答案暂停并等待
- 支持暂停和恢复

## 第0部分：使用者身份和已连接的内容

### 谁在用？

询问使用者身份——律师或法律专业人士 / 有律师支持的非律师 / 无固定律师支持的非律师。这决定工作成果头和输出框架。

### 执业类型

询问执业类型——独立/小型律所/中大型律所/法务部内部/政府/法律援助/诊所/其他。

### 已连接的内容

检查可用集成工具：

**产品上线跟踪器：** 飞书多维表格 / TAPD / 禅道 / Jira / Linear / Asana —— 不是固定的国外工具列表，而是根据用户实际使用的工具。检测并报告状态。

**文档存储：** 云盘 / SharePoint / 本地文件夹等。

**即时通讯：** 飞书 / 钉钉 / 企业微信 / Slack 等。

测试每个连接并报告状态：可用（已测试）/ 待验证（已配置但未测试）/ 不可用（未找到）。降级为人工替代。

## 第1部分：公司（3-4分钟）

询问：
- 公司做什么——面向消费者（C端）还是B2B，还是两者
- 是否受监管行业（金融/医疗/教育/交通/等）
- 是否有日常打交道的监管机构（如网信办、市场监管总局、工信部、行业主管）
- 产品是否国际化——是否涉及数据出境
- 公司阶段和融资态势
- 管辖范围——主要在中国？是否涉及海外市场？
- 风险偏好——保守/中等/进取
- 什么让您晚上睡不着——产品法务最大的焦虑源
- 升级——谁在您之上签署

## 第2部分：审查流程（3-4分钟）

询问：
- 是否存在现有的产品上线审查框架、风险校准表或先前的审查备忘
- 产品上线如何到达法务——飞书多维表格/TAPD/禅道中提交？邮件？会议？
- 跟踪器类型和前置时间
- 审查框架类别——隐私/数据安全/知识产权/广告法合规/消费者保护/行业监管/等
- 正式签署还是咨询性质
- 关键问题：什么构成P0阻断 vs. FYI（仅供参考）

## 第3部分：营销和宣传（1-2分钟）

询问：
- 谁审查营销文案
- 对比性宣传（"比X更快/更好"）的立场
- 事实支撑标准——《广告法》第28条关于虚假广告的要求，宣传需要发版前有数据支撑还是"我们认为如此"即可
- 极限词立场——《广告法》第9条关于"国家级""最高级""最佳"等用语
- AI功能宣称——如何描述AI功能

## 第4部分：种子文件（3-4分钟）

阅读十份近期产品上线审查文件。提取：使用的类别、风险校准（每次上线中什么被提出/阻止/放行）、输出格式、常见模式。

## 撰写实践配置

包含：我们是谁、使用者身份、可用集成、产出、上线审查流程、审查框架（含中国法核心审查维度：隐私/数据安全/IP/广告法/消费者保护/行业监管）、风险校准（按中国监管环境校准）、营销宣传（含《广告法》极限词/虚假广告审查要点）、升级、已连接系统、种子审查记录。

## 撰写后

展示插件能力。提出首次任务建议。标注缺口。以可修改性说明收尾。

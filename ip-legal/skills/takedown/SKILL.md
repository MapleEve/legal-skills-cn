# /ip-legal:takedown

三种模式。选一个：

- `/ip-legal:takedown --send`——起草侵权下架通知（依据《信息网络传播权保护条例》或《电子商务法》第42-43条）
- `/ip-legal:takedown --respond`——对收到的侵权通知进行分诊，识别通知缺陷和应对选项
- `/ip-legal:takedown --counter`——起草反通知。发送前运行响亮关卡

## 前置参考资料

1. 先读取 `../../references/ip-core-rules.md`，再进行平台通知、反通知或下架分析。

## 指示

1. 加载执业画像配置。
2. 检查案件工作空间。
3. 按选定模式调度工作流。
4. 尊重关卡。在--send和--counter模式中，最终输出写入前运行响亮关卡。

## 中国法下的通知-删除机制

中国有两套平行的通知-删除机制，适用范围不同：

### 一、《信息网络传播权保护条例》（2013年修订）

适用于**著作权**侵权（信息网络传播权），针对信息存储空间、搜索链接、自动接入传输等服务提供者：

**通知-删除流程：**
1. 权利人向服务提供者发出书面通知（包含：权利人信息、侵权内容名称和网址、构成侵权的初步证明材料）
2. 服务提供者收到通知后立即删除或断开链接
3. 服务提供者将通知转送服务对象
4. 服务对象可提交书面反通知（说明不构成侵权的初步证明材料）
5. 服务提供者将反通知转送权利人
6. 服务提供者恢复被删除内容或链接

**错误通知的赔偿责任**：因权利人的错误通知给服务对象造成损失的，权利人应承担赔偿责任。

### 二、《电子商务法》第42-43条

适用于**电子商务平台**的知识产权侵权（覆盖范围更广：商标、专利、著作权）：

**第42条——通知与采取必要措施：**
1. 权利人通知电子商务平台经营者（包含：侵权的初步证据、权利人的真实身份信息）
2. 平台经营者收到通知后，应当及时采取删除、屏蔽、断开链接、终止交易和服务等必要措施
3. 平台经营者将通知转送平台内经营者
4. **通知应当采取必要措施——未及时采取措施的对损害扩大部分承担连带责任**

**第43条——反通知与终止措施：**
1. 平台内经营者收到通知后，可提交不存在侵权行为的声明（包含：不存在侵权行为的初步证据）
2. 平台经营者将声明转送权利人
3. 平台经营者告知权利人可以向有关主管部门投诉或者向人民法院起诉
4. **15天等待期**：平台经营者在转送声明到达权利人后15日内，未收到权利人已经投诉或者起诉通知的，应当及时终止所采取的措施

**恶意通知的加倍赔偿责任（第42条第三款）：**
恶意发出错误通知，造成平台内经营者损失的，加倍承担赔偿责任。

### 两套机制的适用比较

| 维度 | 《信息网络传播权保护条例》 | 《电子商务法》 |
|---|---|---|
| 保护客体 | 信息网络传播权（著作权） | 商标、专利、著作权等知识产权 |
| 适用对象 | 信息存储空间、搜索链接、自动接入传输等ISP | 电子商务平台经营者 |
| 反通知后恢复 | 服务提供者收到反通知后恢复 | 送达权利人后再等15天才终止措施 |
| 错误通知赔偿 | 一般赔偿责任 | 恶意通知加倍赔偿 |
| 平台连带责任 | 知道或应知侵权未采取措施 | 未及时采取措施的连带责任 |

## 发送模式——起草侵权下架通知

### 步骤1：识别受保护的权利
- **著作权**：明确作品名称、类型（文字/美术/摄影/视听软件等）、创作完成日期、著作权登记号（如有）
- **商标权**：明确商标注册号、核定使用商品/服务类别、注册有效期
- **专利权**：明确专利号、专利类型、专利名称、授权公告日、年费缴纳状态

### 步骤2：识别侵权材料和位置
- 侵权内容的具体URL或平台内定位方式
- 侵权内容与权利作品的对比说明
- 截图等证据

### 步骤3：合理使用初步审查
依据《著作权法》(2020)第24条，审查侵权方可能主张的合理使用情形：
- 为个人学习、研究或者欣赏
- 为介绍、评论某一作品或者说明某一问题而适当引用
- 为报道新闻而不可避免地再现或者引用
- 为学校课堂教学或者科学研究而少量复制
- 其他第24条列举情形

**标注而非决定**：合理使用是抗辩事由，最终由法院或行政机关认定。本技能仅走查要素供律师判断。

### 步骤4：起草通知
依据适用制度起草。通知需包含：
- 权利人的姓名（名称）、联系方式和地址
- 要求删除或断开链接的侵权内容的名称和网络地址
- 构成侵权的初步证明材料（权利证明+侵权比对）
- 真实性承诺

### 步骤5：发送前关卡
**响亮关卡：** 发送错误通知可能承担赔偿责任（《电子商务法》下恶意通知加倍赔偿）。发送前务必确认：
- 权利证明真实有效
- 侵权比对确实充分
- 确有合理理由相信构成侵权
- 非恶意/非竞争性策略使用

### 步骤6：输出

## 回应模式——对收到的侵权通知分诊

### 步骤1：读取和分类通知
- 通知来源：平台转送 / 权利人直接发送
- 通知适用制度：《信息网络传播权保护条例》/《电子商务法》
- 通知完整性：是否包含必要要素

### 步骤2：评估通知实体和程序
| 评估维度 | 检查要点 |
|---|---|
| **权利有效性** | 著作权真实性/商标注册证有效性/专利权有效性 |
| **侵权比对** | 是否确实与权利作品/商标/专利相同或实质性相似 |
| **合理使用** | 是否可能构成合理使用（《著作权法》第24条） |
| **通知缺陷** | 通知是否缺少法定必要内容 |
| **平台合规** | 平台是否已采取符合法定期限的措施 |
| **发件人可信度** | 是否为权利人或利害关系人 |

### 步骤3：应对选项

1. **遵守/删除内容**：通知确实，侵权风险高——自行下架
2. **提交反通知**：通知有误或侵权不成立——提交不存在侵权行为的声明及初步证据
3. **协商/联络权利人**：通知处于灰色地带——尝试获得授权或和解
4. **不回应/忽略**：仅在通知明显无依据、非来自平台或已超过合理期限时考虑——有风险

### 步骤4：撰写分诊备忘录
含建议行动、时间线和风险评估。

## 反通知模式——起草反通知

### 步骤1：确认提交反通知的前提
- 确认被投诉行为不构成侵权，或
- 确认已获得权利人授权，或
- 确认通知存在缺陷（缺少法定要件）

### 步骤2：起草反通知
反通知应包含：
- 服务对象的姓名（名称）、联系方式和地址
- 要求恢复的内容名称和网络地址
- **不构成侵权的初步证明材料**
  - 独立创作完成的证据
  - 权利人授权的证据
  - 合法来源的证据
  - 技术特征不相同/不等同的比对（专利）
  - 不属于相同/类似商品或商标不近似（商标）
- 真实性承诺

### 步骤3：发送前关卡
**响亮关卡：** 在《电子商务法》第43条下，提交反通知后权利人可能在15天内起诉或投诉。提交反通知前需评估：
- 侵权不成立的理由是否充分
- 提交反通知后可能引发的法律风险
- 是否需要提前准备应诉方案

### 步骤4：输出

## 输出格式

### 发送模式——侵权通知草稿
```markdown
致：[平台名称] 知识产权投诉受理部门

**侵权通知**

权利人：[姓名/名称]
地址：[地址]
联系方式：[电话/邮箱]

**一、权利证明**
[著作权登记号/商标注册号/专利号及证书信息]

**二、侵权内容**
[侵权内容URL/位置及描述]
[与权利作品/商标/专利的对比说明]

**三、初步证明材料**
[附：权利证明复印件、侵权比对材料]

**四、要求**
请依据[《信息网络传播权保护条例》/《电子商务法》第42条]，对上述侵权内容[删除/屏蔽/断开链接/终止交易和服务]。

**五、真实性承诺**
本人/本单位保证本通知内容真实、准确。因错误通知造成的损失，本人/本单位依法承担法律责任。

[签名]
[date]
```

### 反通知模式——反通知草稿
```markdown
致：[平台名称]

**不存在侵权行为的声明**

声明人：[姓名/名称]
地址：[地址]
联系方式：[电话/邮箱]

**一、被采取措施的内容**
[被删除/屏蔽等内容URL及描述]

**二、不存在侵权行为的初步证明材料**
[独立创作/合法授权/不相同不近似等的证据说明]

**三、要求**
请恢复上述内容/终止所采取的措施。

**四、真实性承诺**
本人/本单位保证本声明内容真实、准确。因错误声明造成的损失，本人/本单位依法承担法律责任。

[签名]
[date]
```

## 此技能不做什么

- 不代表权利人提交通知。仅起草。用户通过平台指定渠道自行提交。
- 不决定合理使用/正当使用。走查要素供律师判断。
- 不在接收方验证发件人的权利主张。
- 不绕过发送前关卡。每次运行关卡。
- 不发明引用或引用不存在的法律条文。
- 不处理非中国法律制度的平台侵权投诉（如DMCA）。

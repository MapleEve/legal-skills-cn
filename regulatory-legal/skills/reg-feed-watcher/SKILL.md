# 监管动态源监控器

## 前置参考资料

1. 先读取 `../../references/admin-law-core.md`，再进行监管动态识别、规则层级判断或政策摘要。

## 中国监管体系背景

本技能工作在中国法律监管框架下，遵循以下效力层级：

```
法律（全国人大及其常委会制定）
  └─ 行政法规（国务院制定）
       └─ 地方性法规（省级/设区的市人大制定）
            └─ 部门规章（国务院各部委制定）
                 └─ 规范性文件（各级行政机关）
                      └─ 司法解释（最高人民法院/最高人民检察院）
```

关键监管机构：国务院、市场监管总局、网信办、证监会、金融监管总局、工信部、药监局、生态环境部及各地方对应机构。

企业合规管理遵循国资委《中央企业合规管理办法》框架：合规义务识别 → 合规风险评估 → 合规整改 → 合规报告 → 持续监控。

本技能聚焦于"合规义务识别"与"持续监控"环节。

---

## 目的

从各级监管源头拉取新增法规/规章/规范性文件，按企业重大性标准分级过滤。未经过滤的监管信息是噪音；分级过滤是核心价值。

## 加载上下文

`~/.claude/plugins/config/claude-for-legal-cn/regulatory-legal/CLAUDE.md` → 监管关注清单、重大性标准、动态源配置、摘要输出路径。

`references/source-catalog.md`（本技能目录中）→ 涵盖国务院各部委、地方监管机构、行业自律组织以及国际/二级聚合来源的精选目录。

## 工作流

### 步骤0：覆盖检查（拉取前）

将 CLAUDE.md 中的监管关注清单 + 动态源配置与 `references/source-catalog.md` 对比：

- 按监管关注清单，企业关注哪些层级（国务院/部委/地方监管/行业自律/国际）？
- 关注清单中哪些监管机构配置了零个或极少动态源？

如存在明显缺口（如关心"网信办"但仅配置了单一RSS，缺少地方网信办的重要公告渠道），在摘要顶部展示：

> **发现覆盖缺口：** 你的监管关注清单包含[类别]，但仅配置了[N]个动态源。来源目录在此类别下列出了[X]个选项。需要建议添加吗？运行 `冷启动访谈 --redo` 更新配置。

不重复提示已跳过的缺口；在 CLAUDE.md 中注明状态使其持久。

### 步骤1：拉取

从已配置的动态源层级拉取。免费动态源为基线（始终激活）；付费动态源为增强层（如已配置则使用）。

**一级 — 免费动态源（始终激活）**

对监管关注清单中的每个机构：

- **国务院政策文件库** — 按发文机关、日期范围（自上次检查）、文件类型查询。返回结构化数据：文件类型、标题、摘要、生效日期、征求意见截止日期（针对征求意见稿）和引用。涵盖所有国务院部委级机构。
- **直接监管机构 RSS / 官网公告栏** — 获取并解析 CLAUDE.md 动态源配置中的 RSS URL 或官网公告页面。

常用监管机构动态源参考：

| 监管机构 | 来源/方式 |
|---|---|
| 国务院 | 中国政府网 (www.gov.cn)、国务院政策文件库 |
| 全国人大 | 国家法律法规数据库 (flk.npc.gov.cn)、中国人大网 (www.npc.gov.cn) |
| 市场监管总局 | 市场监管总局官网 (www.samr.gov.cn) |
| 网信办 | 国家互联网信息办公室官网 (www.cac.gov.cn) |
| 证监会 | 中国证监会官网 (www.csrc.gov.cn) |
| 金融监管总局 | 国家金融监督管理总局官网 (www.nfra.gov.cn) |
| 工信部 | 工业和信息化部官网 (www.miit.gov.cn) |
| 药监局 | 国家药品监督管理局官网 (www.nmpa.gov.cn) |
| 生态环境部 | 生态环境部官网 (www.mee.gov.cn) |
| 商务部 | 商务部官网 (www.mofcom.gov.cn) |
| 最高人民法院 | 最高人民法院官网 (www.court.gov.cn)、中国裁判文书网 (wenshu.court.gov.cn) |

对于不在此列表中的监管机构：检查其官方网站获取正确动态源。

**二级 — 省级/地方监管机构**

对关注清单中的地方监管机构，按上述方式拉取其官网公告栏。地方机构可能没有结构化API；降级为官网手动核对或 RSS。

**三级 — 付费动态源（如已配置）**

- **北大法宝 MCP：** 查询自上次检查以来的更新，按监管关注清单中的机构过滤。
- **威科先行 MCP：** 同上。

跨层级去重 — 同一文件可能出现在多个来源中。对富化后的输出优先使用最丰富的来源。

**来源归属。** 标记每个引用和监管项目附其来源：`[国务院政策文件库]`、`[<regulator> RSS]`、`[北大法宝]`、`[威科先行]`、`[网页搜索 — 核实]`、`[模型知识 — 核实]`、`[用户提供]`。标记`核实`的项目虚构风险高于工具获取项目，应首先检查。绝不可剥离或折叠标签。

**二级来源。** 从行业媒体、律所评论、咨询机构报告等二级来源拉取的项目，额外标记为`[二级来源]`。在摘要中注明："→ 追溯至一级来源：[监管机构官网链接，或'在发文机关官网查找核实后再依赖']。"

**四级 — 手动录入**

如用户粘贴监管文本或摘要：将粘贴内容视为单一项目，跳到步骤2进行分类，来源记录为"手动录入"。

拉取后记录检查时间戳。下次运行从此时间戳向前拉取。

### 步骤2：分类

每项按 CLAUDE.md 中的重大性标准分级：

| 项目类型 | 重大性 |
|---|---|
| 正式发布的法律/行政法规/部门规章/司法解释 | 通常"红线"或"重大" |
| 正式发布的规范性文件 | 通常"重大"或"值得审查" |
| 征求意见稿（行政法规/部门规章/规范性文件） | 通常"值得审查" — 且始终记录公开征求意见截止日期 |
| 预征求意见通知 / 立项公告 | 值得审查 — 不改变法律但释放监管方向信号，记录参与截止日期。无合规差距，仅路由至政策差异对比做预判分析。 |
| 信息征集 | 同上 |
| 执法行动 / 行政处罚 | 涉及同行或相关业务领域 → 重大/值得审查；其他 → 例行了解或跳过 |
| 指导性文件 | 值得审查 |
| 领导讲话/声明 | 按阈值例行了解或跳过 |
| 行业自律规则 | 视行业参与程度而定 |

**征求意见稿截止日期处理：**
- 提取公开征求意见截止日期
- 如 CLAUDE.md 中征求意见跟踪已启用：追加到 `comment-tracker.yaml`，状态"未决定"
- 在输出条目中包含截止日期

### 步骤3：富化

对"例行了解"层级以上的每项：
- 一句话摘要（什么变了）
- 为什么对企业在可能有影响（与业务的相关性）
- 来源链接
- 生效日期或征求意见截止日期

不单独摘录"例行了解"项目 — 仅计数。

## 移交

- **至政策差异对比：** 任何具有可能政策影响的"红线"或"重大"项目 → 提供运行差异对比。
- **至合规差距提示器：** 差异对比发现合规差距 → 已跟踪。
- **至征求意见跟踪器：** 征求意见稿 → 征求意见截止日期自动记录。

## 本技能不做的事

- 完整阅读每项内容。分类和富化；深度阅读留给通过过滤器的项目。
- 更改重大性标准。如需调整，编辑 CLAUDE.md。
- 要求付费数据源。免费动态源是基线。

# 尽调表格审查

## 目的

您有一堆文件和一组需要跨每份文件一致回答的问题。输出是一张表：文档行、数据点列，每个单元格可追溯到源文档中的确切词句。

这不是问题发现。`尽调问题提取`在读2,000份文件后找到30个问题。此技能回答所有2,000份文件的相同15个问题。两者都是合法的；它们回答不同的问题。

这也不是人类阅读文档的替代品。此技能产生的每个单元格是一个需要验证的线索，而非一个发现。

## 中国尽调清单分类体系

每一份尽调表格按以下中国法律实务中的尽调分类进行组织：

| 类别编号 | 类别名称 | 主要内容 |
|---|---|---|
| 1 | 主体资格与历史沿革 | 营业执照、公司章程、工商登记档案、股权结构、历次变更登记、出资证明、验资报告 |
| 2 | 股东与股权 | 股东名册、股权代持协议（如有）、股权质押/冻结、优先购买权、对赌协议、一致行动协议 |
| 3 | 组织机构与公司治理 | 股东会/董事会/监事会决议、董事/监事/高管名单及任职资格、三会议事规则、法定代表人信息 |
| 4 | 资产 | 不动产（土地/房产——土地使用权证/不动产权证、租赁合同）、动产（设备/车辆清单及权属证明）、在建工程 |
| 5 | 重大合同 | 供应商/客户合同（交易额前N大）、贷款合同/担保合同、合作协议、合资合同、授权/许可合同 |
| 6 | 知识产权 | 商标/专利/著作权/域名清单及权属证明、许可/被许可合同、知识产权争议、核心技术来源 |
| 7 | 劳动人事 | 员工名册、劳动合同签订率、社保/公积金缴纳情况、未决劳动争议、竞业限制/保密协议、劳务派遣/外包 |
| 8 | 税务 | 税务登记、纳税申报表、完税证明、税收优惠/减免文件、税务稽查/处罚记录、关联交易定价 |
| 9 | 环境保护 | 环评批复/验收文件、排污许可证、危险废物处置、环保处罚记录 |
| 10 | 诉讼与仲裁 | 未决/潜在诉讼仲裁案件清单、行政处罚、财产保全/强制执行 |
| 11 | 行业监管 | 行业许可证/资质证照、行业监管处罚、行业准入条件满足情况 |

## 列类型系统

每列有一个类型约束答案格式：

| 类型 | 说明 | 示例 |
|---|---|---|
| `verbatim` | 逐字引用源文档 | "根据第十条第3款，乙方不得在合同期内..." |
| `classify` | 从固定列表中选择一个值 | `有` / `无` / `不明确` |
| `date` | ISO日期格式 | `2024-01-15` |
| `duration` | 数字+单位 | `3年`、`6个月`、`无限期` |
| `currency` | 数字+币种代码 | `5,000,000 CNY` |
| `number` | 纯数字 | `15`、`200` |
| `free` | 简短自由文本摘要（谨慎使用） | 2-3句话的摘要 |

## "未找到"的三种状态

空白单元格隐藏信息。强制使用三种显式状态之一：

| 状态 | 含义 |
|---|---|
| `not_present` | 确信该主题未被涉及——文件已完整审查，确认不含该信息 |
| `unclear` | 存在相关内容但无法自信分类——语言模糊/有歧义 |
| `needs_review` | 找到内容但必须由律师决定——涉及法律判断 |

## 工作流

### 步骤0：确认文件来源和输出格式

先读取本技能目录下的打包参考材料：
- `references/ma-diligence-columns.md` — M&A 尽调默认字段集；后续 `review-schema.yaml` 以此为起点，再按交易文件和用户目标增删字段。
- 若用户选择 Excel 输出，先读取 `references/excel-output.md`，并按其中的 workbook 结构、批注、状态颜色和公式注入防护生成。
- 若用户选择 Google Sheets 输出，先读取 `references/gsheets-output.md`，并按其中的 spreadsheet 创建、notes、分享限制和公式注入防护生成。

确认：
- 文件来源（数据室路径、本地文件夹、或粘贴的文档列表）
- 尽调类别范围（全部11个类别或仅特定类别）
- 输出格式（Markdown表格 + CSV + Excel，或用户明确选择 Google Sheets）

### 步骤1：构建并确认审查模式

为每份文档类型生成 `review-schema.yaml`：

```yaml
# 审查模式：[合同类型名称]
适用类别: "[尽调类别编号和名称]"
审查字段:
  - column_id: "1"
    column_name: "[字段名]"
    column_type: "[verbatim/classify/date/duration/currency/number/free]"
    question: "[每份文档需要回答的具体问题]"
    classify_options: ["有", "无", "不明确"]  # 仅classify类型
    chinese_law_ref: "[相关中国法律法规条文——如适用]"
```

以**重大合同类别**为例：

```yaml
# 审查模式：重大合同
适用类别: "5——重大合同"
审查字段:
  - column_id: "1"
    column_name: "合同名称"
    column_type: "verbatim"
    question: "合同全称是什么？"

  - column_id: "2"
    column_name: "合同相对方"
    column_type: "verbatim"
    question: "与谁签订？"

  - column_id: "3"
    column_name: "签署日期"
    column_type: "date"
    question: "合同何时签署？"

  - column_id: "4"
    column_name: "合同期限"
    column_type: "duration"
    question: "合同有效期限多长？"

  - column_id: "5"
    column_name: "控制权变更条款"
    column_type: "classify"
    question: "是否包含控制权变更（Change of Control）条款？"
    classify_options: ["有", "无", "不明确"]

  - column_id: "6"
    column_name: "控制权变更条款内容"
    column_type: "verbatim"
    question: "控制权变更条款的具体内容是什么？（如上一列为"有"）"

  - column_id: "7"
    column_name: "转让限制"
    column_type: "classify"
    question: "是否包含合同权利义务转让的限制条款？"
    classify_options: ["有——需对方同意", "有——禁止转让", "无限制", "不明确"]

  - column_id: "8"
    column_name: "终止权"
    column_type: "classify"
    question: "合同是否赋予任一方单方终止权？"
    classify_options: ["任一方可随时终止", "特定条件下可终止", "无单方终止权", "不明确"]

  - column_id: "9"
    column_name: "排他性条款"
    column_type: "classify"
    question: "是否包含排他性/不竞争条款？"
    classify_options: ["有", "无", "不明确"]

  - column_id: "10"
    column_name: "最惠国待遇条款"
    column_type: "classify"
    question: "是否包含最惠国待遇（MFN）条款？"
    classify_options: ["有", "无", "不明确"]

  - column_id: "11"
    column_name: "违约责任"
    column_type: "free"
    question: "违约责任条款的主要内容？（违约金金额、责任上限等）"

  - column_id: "12"
    column_name: "争议解决"
    column_type: "classify"
    question: "争议解决方式是什么？"
    classify_options: ["诉讼——有管辖约定", "诉讼——无管辖约定", "仲裁——有仲裁机构", "仲裁——无仲裁机构", "不明确"]

  - column_id: "13"
    column_name: "担保/保证"
    column_type: "classify"
    question: "是否包含担保或保证条款？"
    classify_options: ["有", "无", "不明确"]

  - column_id: "14"
    column_name: "合同金额"
    column_type: "currency"
    question: "合同总金额/年金额？"

  - column_id: "15"
    column_name: "备注"
    column_type: "free"
    question: "需要律师关注的任何其他事项"
```

### 步骤2：样本运行（3-5份文档）

选择3-5份代表性文档运行审查。验证：
- 审查模式是否合理？字段是否够用或过多？
- 三种"未找到"状态使用是否恰当？
- 逐字引用是否正确截取？

向律师展示样本结果，获得签字确认后继续。

### 步骤3：并行展开

对全部文档并行运行审查（每文档一个子代理）。

### 步骤4：规范化

审查完成后，对整个表格逐列检查：
- 同一列中 `classify` 的值是否使用一致？
- 是否有空白单元格应标记为 `not_present` / `unclear` / `needs_review`？
- 逐字引用是否正确标注了源文档及条款编号？

### 步骤5：输出

输出三种格式：

1. **Markdown表格**（便于在备忘录中嵌入）
2. **CSV文件**（便于进一步处理）
3. **Excel文件**（便于交付客户/交易对手——含冻结首行、筛选器、类别分Sheet）

### 步骤6：摘要

生成审查摘要，包含：
- 审查了多少份文档
- 各"未找到"状态的数量统计
- 需要律师立即关注的项目（如所有 `needs_review` 行）
- 数据完整度（各列的覆盖比例）

## 此技能不做什么

- 不替代阅读文档。告诉您去哪里看。
- 不产生置信度分数。`unclear`/`needs_review` 状态和逐字引用本身就是置信度信号。
- 不静默跳过文档。无法读取的文档获得一行 `needs_review` 附备注。
- 不假装释义就是引用。逐字引用可溯源是核心要点。

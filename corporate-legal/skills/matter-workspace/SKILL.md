# 事项工作区

## 中国公司法务执业场景

中国公司法务/外部律师常同时处理多个事项：
- **外部律师** — 为多家企业客户提供常年法律顾问或专项服务
- **企业集团法务** — 为集团内多家子公司/业务线提供法律服务
- **投资机构法务** — 管理已投项目的投后法律事务

事项工作区确保每个客户/项目的数据隔离，符合：
- 《律师法》保密义务要求
- 《民法典》合同编保密条款
- 企业内部数据管理要求

## 存储布局

所有事项数据位于 `~/.claude/plugins/config/claude-for-legal-cn/corporate-legal/matters/` 下：

```
matters/
├── <slug>/
│   ├── matter.md               # 客户、事项类型、关键事实
│   ├── history.md              # 事件日志
│   ├── notes.md                # 工作笔记
│   └── outputs/                # 该事项的技能输出
└── _archived/
    └── <slug>/                 # 已关闭事项 — 可读但非活跃
```

Slug 使用小写连字符。示例：`alibaba-acquisition-2026`、`tencent-jv-structure`、`baidu-data-room`。

## 当前事项

执业级 CLAUDE.md 中 `## 事项工作区` 下的 `当前事项：` 行是唯一真相来源。

## 子命令

### `new <slug>`
创建新事项工作区。中国投资并购事项类型包括：
- **投资并购买方** — 收购目标公司股权/资产
- **投资并购卖方** — 出售子公司/业务线
- **融资** — 被投企业融资轮次
- **合资企业** — 与其他方设立JV
- **资产重组** — 集团内部资产/业务重组
- **IPO/上市准备** — 上市前合规梳理
- **常法顾问** — 持续性的常年法律顾问服务
- **争议解决** — 诉讼/仲裁事项
- **投后管理** — 已投项目的持续法律事务

信息采集：客户、对方当事人、事项类型、保密级别、关键事实、事项特定覆盖。

### `list`
枚举所有事项，打印表格，标注当前事项。已归档事项单独列出。

### `switch <slug>`
编辑当前事项行。展示事项摘要以确认。

### `close <slug>`
归档事项 — 移至 `_archived`（永不删除），记录关闭日期。如关闭的为当前事项，设置为仅执业级。

### `none`
设置为仅执业级上下文。

## 跨事项上下文

默认为关。当为关时，在一个事项中工作的技能绝不读取另一事项的文件。当为开时，仅在用户明确要求时读取跨事项文件（如："比较我们最近五个交易中关于责任上限的条款"）。

## 中国执业特别注意

- 利益冲突检查是本技能不覆盖的领域 — 需由执业的律师/律所人工完成
- 事项文件中的法律分析和建议需考虑律师保密义务、委托合同保密、商业秘密/个人信息保护，以及法院、仲裁、监管或刑事调取风险
- 对方当事人的保密信息（如尽调中获取的资料）需在事项关闭时按保密协议处理

## 此技能不做什么

- 不运行利益冲突检查
- 不执行文件保留政策
- 不自动路由输出
- 不决定跨事项是否适当 — 读取标志并服从
